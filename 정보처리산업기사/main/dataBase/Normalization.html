정규화

  • 관계형 스키마를 더 작은 속성의 세트로 쪼개어 이상 현상을 줄여나가는 과정
  • 중복성및 종속성을 배제시키는 방법

이상(Anomaly)의 종류

  • 삽입 이상(Insertion Anomaly) : 릴레이션에 데이터를 삽입할 때 의도와 상관없이 원하지 않는 값 들도 함께 삽입되는 현상
  • 삭제 이상(Deletion Anomaly) : 릴레이션에서 한 튜플을 삭제할 때 의도와 상관없는 값들도 함께 삭제되는 연쇄 삭제 현상이 일어나는 현상
  • 갱신 이상(Update Anomaly) : 릴레이션에서 튜플에 있는 속성값을 갱신할 때 일부 튜플의 정보 만 갱신되어 정보에 모순이 생기는 현상

정규화 과정

  • 1NF로 정규화 : 릴레이션에 속한 모든 도메인이 원자값으로 되어있는 릴레이션
  • 2NF로 정규화 : 1NF를 만족하고 키가 아닌 모든 속성이 기본키에 대하여 완전 함수적 종속 관계를 만족한다.
  • 3NF로 정규화 : 2NF를 만족하고 키가 아닌 모든 속성이 기본키에 대해 이행적 종속 관계 를 이루지 않도록 제한한 관계형
  • 비정규 릴레이션
  • -도메인이 원자값이되면
  • 1NF로 정규화
  • -부분적 함수 종속을 제거하면
  • 2NF로 정규화
  • -이행적 함수 종속을 제거하면
  • 3NF로 정규화
  • -결정자이면서 후보키가 아닌 것 제거
  • BCNF로 정규화
  • -다치 종속
  • 4NF로 정규화
  • -조인 종속성 이용
  • 5NF로 정규화
  • 함수적 종속 관계
    : EX) 수강 릴레이션 (학번, 이름, 과목명)으로 되어 있을때 학번이 결정되면 과목명에 관계없이 학번에 항상 같은 이름이 대응된다. 이를 이름을 학번에 함수 종속적이라고 하며 학번 -> 이름 과 같이 작성한다. 이때 학번을 결정자 이름을 종속자 라고 한다.
  • 완전 함수적 종속 관계
    : 속성 A가 다른 속성들의 집합 B전체에 대해서 함수적 종속 관계를 갖지만 집합 B의 진부분집합에 대해서는 종속관계를 갖지 않으면, 속성A는 집합B의 속성들에 대해 완전 함수적 종속 관계에 있다고 한다.
    EX) 수강 릴레이션 (학번, 성적, 과목명)으로 되어 있을때 성적은 (학번, 과목명)에 함수 종속적 이지만 학번과 과목명 각각에 대해서는 함수 종속이 아니다.
  • 이행적 함수 종속
    : 기본키 이외의 다른 속성이 그외의 다른 속성을 결정 할 수 있을때를 말한다.
    EX)학생 테이블(학번, 주민등록번호, 생년월일, 이름) 이 있을때 학번이 기본키 라고 할때 주민등록번호는 기본키가 아님에도 불구하고 생년월일과 이름을 결정 할 수 있다. 이때 (학번, 주민등록번호) (주민등록번호, 생년월일, 이름)과 같이 두개의 테이블로 나누어 이행적 함수 종속을 없앨 수 있다.
    책에서는 A->B 이고 B->C 일 때 A->C를 만족하는 관계라고 한다.
