SQL

  • 국제표준 데이터베이스 언어
  • 정의, 조작, 제어의 기능으로 나누어져 있다.
  • 관계형 데이터베이스의 표준 질의어(Query Language)
    질의어 : 데이터베이스 파일과 범용 프로그래밍 언어를 정확히 알지 못하는 단말 사용자들이 단말기를 통해서 대화식으로 쉽게 DB를 이용할 수 있도록 되어 있는 비절차어의 일종이다.
  • DDL(Data Define Language)정의어 DML(Data Manipulation Language) 조작어 DCL(Data Control Language)제어어 로 나누어져 있다.

1.DDL(Data Define Language)정의어

  • SCHEMA, DOMAIN, TABLE, VIEW, INDEX를 정의하거나 변경 또는 삭제할 떄 사용하는 명령문이다
    DDL로 정의된 내용은 Meta-date가 되며, 시스템 카탈로그에 저장한다.
    Meta-data : 데이터를 관리하기 위한 데이터
    시스템 카탈로그 : 시스템 구성에 관련이 있는 객체들의 정보를 포함하는 시스템 데이터베이스 테이블
CREATE SCHEMA
  • 스키마를 정의하는 명령문
  • 스키마의 식별을 위해 스키마 이름과 소유권자나 허가권자를 정의한다.
  • 표기형식 : CREATE SCHEMA 스키마_이름 AUTHORIZATION 사용자_ID;
CREATE DOMAIN
  • 도메인을 정의하는 명령문
  • 임의의 속성(Attribute)에서 취할 수 있는 원자값의 범위를 일부분으로 제한할 수 있다.
  • 표기형식
    CREATE DOMAIN 도메인_이름 data_type
    [DEFAULT 디폴트값_정의]
    [CONSTRAINT VALID-도메인_이름 CHECK (범위)];
  • 예시
    CREATE SEX CHAR(1)
    DEFAULT '?'
    CONSTRAINT VALID-SEX CHECK(VALUE IN('남','여','?'));
CREATE DOMAIN
  • 공사중

2DML(Data Manipulation Language)조작어

  • UPDATE : 특정 튜플의 내용을 변경시킬때 사용
  • UPDATE 테이블명 SET 변경시킬내용 WHERE 조건; ex) UPDATE 학생 SET 등급 = 'A' WHERE 점수 >= 90;

3. DCL(Data Control Language)제어어

  • 데이터의 사용 권한을 관리하는데 사용하며 GRANT, REVOKE문 등이 있다.
    TCL인 COMMIT, ROLLBACK 까지 포함해서 DCL 이라고 분류하기도 한다.
GRANT문
  • 특정 데이터베이스 사용자에게 특정 작업에 대한 수행 권한을 부여한다.
  • 표기형식
    GRANT 권한 ON 데이터_개체 TO 사용자 [WITH GRANT OPTION];
  • 권한 종류 : ALL, SELECT, INSERT, DELETE, UPDATE, INDEX, ALTER등
  • 사용자 종류 : 사용자 ID : 단순 사용자
    ROLE : 사용자에게 부여할 수 있는 권한들을 묶어서 하나의 이름을 정해 저장해 놓은것으로 쉽게 권한을 관리 할 수 있다.
  • public : 모든 사용자
  • WITH GRANT OPITON : 부여받은 권한을 다른 사용자에게 부여 할 수 있는 권한을 부여함
REVOKE문
  • 특정 데이터베이스 사용자에게 특정 작업에 대한 수행 권한을 박탈 or 회수 한다.
